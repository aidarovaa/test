1. Fetch нужен для получения данных с сервера, fetch использует promise, который позволяет использовать более простой и чистый api. 2. Promise нужен для использования асинхронных запросов, и не сработает пока запрос не будет успешно выполнен. Также имеет три состояния: pending (ожидание), fulfilled - исполнено (выполнено успешно) или rejected - отклонено (выполнено с ошибкой). 3. Контекст js - значение переменой this внутри функции. Область видимости определяет доступ к переменным при вызове функции и является уникальной для каждого вызова. 4. Стрелочная функция это и есть анонимная функция, у стрелочной функции удобный, читабельный и компактный синтаксис. Наследует контекст, так как у него нет собственного контекста, и также нет собственного значения this. 5. Async/await позволяет писать код, который выглядит как синхронный, но используется для решения асинхронных задач и не блокирует главный поток. Await оператор, предназначенный для ожидания разрешения или отклонения promise. Его можно использовать только внутри функции async. Async заставляет функцию возвращать promise.
